# Technical Context & Architectural Blueprint
## Unified Control Intelligence Layer (UCIL)

This document provides the technical context, regulatory background, and system design patterns for the Unified Control Intelligence Layer (UCIL). It serves as the primary engineering reference for maintaining the codebase.

---

## 1. Product Purpose & Target Audience

### 1.1 Purpose
UCIL is designed to consolidate, track, and audit cybersecurity and compliance controls in highly regulated enterprise environments. It serves as a unified compliance registry that deduplicates requirements across standards. 

### 1.2 Target Audience
* **CISO / Board of Directors**: Monitors compliance coverage, audit validation histories, and potential exposure gaps.
* **Compliance Lead / Risk Officers**: Oversees data ingestion, scraper jobs, policy overrides, and triggers remediation reviews.
* **Domain Heads (IT Security, HR, DevOps, IT Operations)**: Acts as the final authority to review controls and approve submitted evidence.
* **Control Owners (System Admins, Security Engineers)**: Uploads logs, config files, and audit trails as compliance evidence.

---

## 2. Regulatory Compliance Domain (India & Global)

UCIL bridges the gap between global frameworks and local mandates:

### 2.1 Global Security Frameworks
* **ISO/IEC 27001 (Information Security Management)**: Standardizes asset management, access controls, cryptography, and operations.
* **NIST Cybersecurity Framework (CSF)**: Structures operational processes across Identify, Protect, Detect, Respond, and Recover profiles.
* **PCI-DSS (Payment Card Industry Data Security Standard)**: Imposes strict network segmentation, database logging, and data transmission standards.

### 2.2 Reserve Bank of India (RBI) Cyber Security Framework
The RBI CSF mandates strict compliance thresholds for Indian financial entities (Banks, NBFCs, and payment gateways):
* **Control Mapping**: Banks must map every operational IT action directly back to RBI circular clauses.
* **Audit Logs and Tracing**: Systems must record all user sessions and administrative adjustments.
* **Data Residency**: Customer financial data and compliance audit trails must reside locally within Indian geographic boundaries.

## 3. Database-First Single Source of Truth (SSoT)

The platform enforces a strict database-first architectural pattern. Client applications are lightweight rendering layouts, while the database manages business rules:

```
[ Frontend Client ] 
        │ (Reads from DB View)
        ▼
[ v_framework_coverage / v_controls_full ] 
        │ (Computes aggregations dynamically)
        ▼
[ Controls / Mappings / Evidence Tables ] ──◄── [ DB Trigger recalculate_metrics() ]
```

### 3.1 Views instead of Client-Side Loops
Instead of parsing complex join statements in client code, the UI queries pre-computed database views:
* **`v_framework_coverage`**: Joins `frameworks` and `control_framework_mappings` to compute coverage percentages dynamically.
* **`v_controls_full`**: Combines controls with their domains, owners, head details, and logs.
* **`v_evidence_full`**: Joins uploaded evidence files with control requirements and user names.

---

## 4. Row-Level Security (RLS) & Storage Access

* **Least Privilege**: Row-Level Security is active on all tables. Queries filter results based on the logged-in user's role and domain assignment (`public.get_user_role()`).
* **Storage Isolation**: The `evidence-files` bucket is private. The frontend downloads files using signed URLs generated by the API (`createSignedUrl`), expiring after 3600 seconds.

---

## 5. LLaMA Edge Functions & Prompt Caching

To reduce Groq API limits, prompt templates are cached in the database using a 5-minute Time-To-Live (TTL) mapping inside `supabase/functions/_shared/utils.ts`. 

### 5.1 Fallback Status Codes
When Groq API limits are exhausted (HTTP 429) or Deno Edge Functions timeout:
1. The exception is caught and logged.
2. The record's `ai_status` column is set to `'error'`.
3. The UI page intercepts the state, displaying a fallback card that prompts manual SME review.
